# サイトデプロイ

以下の手順でサイトをデプロイしてください。

## 手順

1. **未コミットの変更を確認**
   - `git status` で変更ファイルを確認
   - 変更がなければ「デプロイする変更がありません」と伝えて終了

2. **HTMLバリデーション**
   - `npx html-validate *.html` でHTML構文をチェック（npxが使えない場合はスキップ可）
   - エラーがあれば内容を表示し、修正するか確認

3. **変更をコミット**
   - 変更内容に応じた日本語コミットメッセージを作成
   - プレフィックス規約に従う: `design:`, `feature:`, `fix:`, `content:`, `config:`, `sns:`, `video:`
   - `git add` で関連ファイルをステージング（.env や秘密情報ファイルは除外）
   - `git commit` を実行

4. **mainブランチにプッシュ**
   - 現在のブランチが `main` でない場合は警告し、mainへのマージ方法を案内
   - `main` ブランチの場合は `git push origin main` を実行

5. **デプロイ完了報告**
   - プッシュ成功を確認
   - GitHub Pagesの自動デプロイが開始されたことを伝える
   - サイトURLを表示

## 注意事項
- `.env`、`*.key`、認証情報は絶対にコミットしない
- 大きな変更の場合はブランチを切ってPRを推奨
