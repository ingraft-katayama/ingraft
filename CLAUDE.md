# ingraft inc. プロジェクト規約

## 会社概要

株式会社ingraft（イングラフト）は、東京を拠点とするデジタルクリエイティブ制作会社です。
このリポジトリは、自社サイト・動画制作・SNS運用の3つの業務をClaude Codeで統合管理するためのワークスペースです。

## チーム構成

- 2〜3名（エンジニア＋非エンジニア混合）
- 全員がClaude Codeを使って作業する
- 技術レベルに関係なく、スキル（後述）を使えば同じ結果を得られる設計

## 新メンバー向けセットアップ手順

### 1. Claude Codeのインストールとログイン

チームプランのアカウントでClaude Codeにログインしてください。

### 2. リポジトリのクローン

```bash
git clone https://github.com/ingraft/ingraft.git
cd ingraft
```

### 3. 環境変数の設定

以下の環境変数を `~/.zshrc` に追加してください（値はチームリーダーから受け取る）：

```bash
# 必須
export GITHUB_PAT="your-github-personal-access-token"

# SNS運用で必要（Phase 2以降）
export TWITTER_API_KEY="..."
export TWITTER_API_SECRET="..."
export TWITTER_ACCESS_TOKEN="..."
export TWITTER_ACCESS_SECRET="..."
export INSTAGRAM_ACCESS_TOKEN="..."
export INSTAGRAM_BUSINESS_ID="..."

# 動画・YouTube（Phase 3以降）
export YOUTUBE_API_KEY="..."
export YOUTUBE_CLIENT_ID="..."
export YOUTUBE_CLIENT_SECRET="..."

# アナリティクス（Phase 2以降）
export GA_PROPERTY_ID="..."
export GA_SERVICE_ACCOUNT_KEY_PATH="..."
```

設定後、ターミナルを再起動するか `source ~/.zshrc` を実行。

### 4. MCPコネクタの有効化

#### CLI版（ターミナル）の場合
初回のみ、以下のMCP接続で認証が求められます：
- **Slack**: OAuthフローでブラウザが開くので、会社のSlackワークスペースで認証
- **GitHub**: 環境変数 `GITHUB_PAT` が設定されていれば自動接続
- **Figma**: Claude Codeが初回接続時に案内します
- `/mcp` コマンドでMCPサーバーの接続状況を確認できます

#### デスクトップアプリ版の場合
組織設定のコネクタとは別に、**個人レベルでのコネクタ有効化**が必要です：
1. 左下のユーザーアイコン → **「設定」** を開く
2. 左ナビから **「コネクタ」** を選択
3. 上部バナーの **「カスタマイズに移動」** をクリック
4. **「コネクタ」** タブで、Slack・Gmail・Google Calendar・Figma 等を個人アカウントで接続

**注意**: 組織設定でコネクタが追加されていても、各メンバーがカスタマイズ画面から個別に接続しないとアプリ版では使えません。

### 5. 動作確認

Claude Codeで以下を試してください：
- 「プロジェクトの概要を教えて」→ このファイルの内容が要約されればOK
- `/deploy` → デプロイスキルが認識されればOK
- 「自分宛のSlack DMにテストメッセージを送って」→ Slack MCPが動作すればOK

## 3つの業務領域

### 1. 自社サイト管理

**ワークフロー:**
1. Figmaでデザインを制作・更新
2. `/figma-to-code` でデザインをコードに変換
3. ローカルプレビューで確認
4. `/deploy` でサイトを公開

**ホスティング:** GitHub Pages（`main`ブランチへのpush時に自動デプロイ）

**アクセス解析:**
- `/analytics-report` でGoogle Analyticsのレポートを生成
- レポートはSlackに通知される（週次スケジュール）

### 2. 動画制作

**ワークフロー:**
1. Google Drive上の台本・素材を準備
2. `/video-edit` で動画編集（FFmpegベース）
3. 出力を人間が確認、修正指示があれば再編集
4. ショート動画への切り出し
5. YouTube/SNSへアップロード

**対応操作:** トリミング、結合、テロップ追加、ウォーターマーク、ショート切り出し

**前提条件:** FFmpegがインストールされていること（`brew install ffmpeg`）

### 3. SNS運用（X / Instagram / YouTube）

**ワークフロー:**
1. `/trend-report` でトレンドを調査
2. `/sns-post` で投稿原稿を作成
3. 原稿は `sns/drafts/` に保存され、Slackに通知
4. **人間がレビュー・承認**（← 必須）
5. 承認後、Claudeが各プラットフォームに投稿

**重要ルール: SNS投稿は必ず人間が確認してから公開すること。**

## セキュリティルール

- **APIキー・トークンは絶対にコミットしない**（環境変数で管理）
- `.env`、`*.key`、認証情報ファイルは `.gitignore` に含まれている
- プロジェクトの `.mcp.json` には秘密情報を直接書かない（`${ENV_VAR}` で参照）
- SNS投稿は必ず人間の承認を経てから公開

## 設定ファイルの仕組み（プロジェクト vs 個人）

Claude Codeには「プロジェクトの設定」と「個人の設定」の2つがあります。

### プロジェクトの設定（チーム全員で共有）

リポジトリの中にあるファイルです。`git clone` すれば自動的に全員に適用されます。

| ファイル | 何が書いてあるか |
|----------|------------------|
| `.claude/settings.json` | Claudeに何を許可するか（ファイル操作、コマンド実行、MCP利用など） |
| `.claude/launch.json` | ローカルプレビューサーバーの設定 |
| `.claude/skills/` | チーム共通のスキル（`/deploy`、`/sns-post` など） |
| `.mcp.json` | 外部サービスとの接続先（GitHub等。APIキーは含まない） |
| `CLAUDE.md` | このファイル。Claudeへの指示書 |

### 個人の設定（自分だけに適用）

自分のパソコンの `~/.claude/` フォルダにあるファイルです。Gitには含まれません。

| ファイル | 何が書いてあるか |
|----------|------------------|
| `~/.claude/settings.json` | 個人用の追加設定（プロジェクト設定を上書きしたい場合など） |
| `~/.zshrc` の環境変数 | APIキーやトークン（`GITHUB_PAT` など） |

### 設定を変更したいとき

**設定ファイルを自分で直接編集しないでください。** 必ずClaudeに相談してください。

1. Claudeに「〇〇の設定を変えたい」と伝える
2. Claudeがその変更を**プロジェクトの設定**に入れるべきか**個人の設定**に入れるべきか判断して提案する
3. 内容を確認してOKを出す
4. Claudeが編集・コミット・プッシュまで行う

こうすることで、設定ミスや秘密情報の漏洩を防ぎ、チーム全員の環境を常に正しい状態に保てます。

### プロジェクトと個人の判断基準

| こういう変更 | どっちに入れるか |
|-------------|-----------------|
| 全員に共通のルールや権限 | プロジェクト |
| 新しいスキルの追加 | プロジェクト |
| MCP接続先の追加・変更 | プロジェクト |
| 自分だけのAPIキーやトークン | 個人（環境変数） |
| 自分だけの追加設定 | 個人 |

**原則: 秘密情報（パスワード、APIキー等）は絶対にプロジェクトの設定に書かない。**

## プロジェクト変更時の自動Slack通知

以下のファイルが `main` ブランチにpushされると、GitHub Actions（`.github/workflows/notify-config-changes.yml`）が自動的に `#claude` チャンネルへ `@channel` メンション付きで通知します。**手動での通知は不要です。**

### 対象ファイル
- `.claude/skills/` 配下のスキル定義ファイル（新規追加・変更・削除）
- `.claude/settings.json`（権限設定の変更）
- `.mcp.json`（MCP接続設定の変更）
- `CLAUDE.md`（プロジェクト規約の変更）
- `.github/workflows/` 配下のワークフロー（CI/CD変更）

### 仕組み
- pushやPRマージなど、mainブランチへの変更経路を問わず自動通知
- 通知にはSlack Incoming Webhookを使用（`SLACK_WEBHOOK_URL` はGitHub Secretsで管理）
- 通知内容: 変更カテゴリ、変更ファイル一覧、変更者、コミットリンク

## ワークツリーでの作業ルール

- ワークツリーセッションで「設定を変更した」「前に作ったファイルを確認して」等、別セッションでの変更を参照する指示があった場合、**カレントワークツリーだけを見て「ありません」と返さないこと**
- まず `git worktree list` でメインリポジトリのパスを取得し、メインリポジトリおよび他のワークツリーを確認する
- ワークツリーはクリーンな状態で作成されるため、別セッションでの変更はそこには存在しない
- **ワークツリーでの作業開始時に `git merge main --no-edit` でmainの最新を取り込むこと**（設定ファイルや共有ルールの反映漏れを防ぐため）。コンフリクトが発生した場合はマージを中断し、ユーザーに報告して判断を仰ぐ
- ワークツリーでmainにマージすべき変更（コミット済み）がある場合、**セッション終了前に必ず「mainにマージしますか？」とユーザーに確認すること**。確認なしにマージせず、また確認なしに放置もしない

## トラブルシューティング

### MCP接続エラー
- 環境変数が正しく設定されているか確認: `echo $GITHUB_PAT`
- Slack OAuthが期限切れの場合は再認証が必要

### デプロイが動かない
- `git status` で未コミットの変更がないか確認
- `main`ブランチにいるか確認
- GitHub Pagesが有効になっているか確認（リポジトリSettings > Pages）

### FFmpegエラー
- インストール確認: `ffmpeg -version`
- 未インストールの場合: `brew install ffmpeg`
